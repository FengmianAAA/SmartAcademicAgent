# 运行部署说明

更新时间：2026-03-23

## 1. 环境要求

### 1.1 开发环境
- Windows 10/11 或其他可运行 Python 3.11、Node.js 20、MySQL 8 的环境
- Python：建议 3.11
- Node.js：建议 20.x
- npm：随 Node.js 安装
- MySQL：建议 8.0，字符集 `utf8mb4`

### 1.2 项目目录
- 前端：`frontend`
- 后端：`backend`
- SQL：`sql\schema.sql`
- 当前进度：`docs\当前开发进度.md`

## 2. 数据库初始化

### 2.1 创建数据库
```sql
CREATE DATABASE smart_academic_system DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.2 导入表结构
```powershell
mysql -u root -p smart_academic_system < sql\schema.sql
```

## 3. 后端启动步骤

### 3.1 创建虚拟环境并安装依赖
```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

### 3.2 配置环境变量
```powershell
Copy-Item .env.example .env
```

默认配置文件路径：`backend\.env`

关键变量如下：
- `MYSQL_HOST=127.0.0.1`
- `MYSQL_PORT=3306`
- `MYSQL_USER=root`
- `MYSQL_PASSWORD=root`
- `MYSQL_DB=smart_academic_system`
- `JWT_SECRET_KEY=change-this-in-production`
- `API_PREFIX=/api/v1`

### 3.3 启动开发服务
```powershell
uvicorn app.main:app --reload
```

启动后访问：
- 接口根地址：`http://127.0.0.1:8000/api/v1`
- OpenAPI 文档：`http://127.0.0.1:8000/docs`

### 3.4 导入演示数据
```powershell
$env:PYTHONPATH='.'
python scripts\seed_demo_data.py
```

导入后可使用演示账号：
- 学生：`student1 / 123456`
- 管理员：`admin1 / 123456`

## 4. 前端启动步骤

### 4.1 安装依赖
```powershell
cd frontend
npm install
```

### 4.2 配置环境变量
```powershell
Copy-Item .env.example .env
```

默认配置文件路径：`frontend\.env`

关键变量如下：
- `VITE_API_BASE_URL=http://127.0.0.1:8000/api/v1`

### 4.3 启动开发服务
```powershell
npm run dev
```

启动后访问：
- 前端页面：`http://127.0.0.1:5173`

## 5. 联调顺序建议

1. 先启动 MySQL 并导入 `sql\schema.sql`。
2. 启动后端并确认 `http://127.0.0.1:8000/docs` 可访问。
3. 运行 `python scripts\seed_demo_data.py` 导入演示数据。
4. 启动前端并打开 `http://127.0.0.1:5173`。
5. 分别用学生账号和管理员账号验证功能。

## 6. 生产部署建议

### 6.1 前端
```powershell
cd frontend
npm run build
```

构建产物目录：`frontend\dist`

部署建议：
- 使用 Nginx 或静态文件服务器托管 `dist`
- 将 `VITE_API_BASE_URL` 指向正式后端地址，例如 `http://server-ip:8000/api/v1`

### 6.2 后端
```powershell
cd backend
.\.venv\Scripts\activate
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

部署建议：
- 使用进程管理器守护 FastAPI 服务
- 生产环境替换 `JWT_SECRET_KEY`
- MySQL 单独部署并做好账号、权限和备份策略
- 通过反向代理统一暴露前后端访问入口

## 7. 系统测试与截图建议

### 7.1 建议保留的截图
- 登录页：学生与管理员共用登录入口
- 学生首页：学分概览、风险等级、微专业推荐
- 成绩页：课程成绩与通过状态
- 智能问答页：中文提问与回答结果
- 知识库页：文档列表、检索结果、管理员编辑态
- 管理员首页：统计卡片、分布图、规则快照、最近预警
- 预警规则页：新增或编辑规则
- 预警记录页：状态更新前后对比

### 7.2 建议测试流程
1. 管理员登录后截图管理首页。
2. 进入预警规则页展示规则表达式与启用状态。
3. 进入预警记录页，将某条记录从 `new` 改为 `resolved`。
4. 返回管理员首页截图，说明处置分布与完成率变化。
5. 切换学生账号，展示首页、成绩、问答与知识库检索结果。

## 8. 常见问题

### 8.1 后端无法连接数据库
检查 `backend\.env` 中的 MySQL 配置是否与本地数据库一致。

### 8.2 前端登录后接口报错
检查 `frontend\.env` 中的 `VITE_API_BASE_URL` 是否与后端实际地址一致。

### 8.3 演示账号无法登录
重新执行：
```powershell
cd backend
.\.venv\Scripts\activate
$env:PYTHONPATH='.'
python scripts\seed_demo_data.py
```
