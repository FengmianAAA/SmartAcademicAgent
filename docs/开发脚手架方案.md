# 开发脚手架方案

## 1. 技术选型

### 1.1 前端
- `Vue 3`
- `TypeScript`
- `Vite`
- `Pinia`
- `Vue Router`
- 后续可补充 `Element Plus` 或 `Naive UI`

### 1.2 后端
- `FastAPI`
- `SQLAlchemy`
- `MySQL`
- `JWT`
- 后续可补充 `Redis`、`Celery`、`FAISS`

## 2. 推荐目录结构

```text
frontend
  src
    api            接口请求封装
    assets         静态资源
    components     通用组件
    router         路由配置
    stores         状态管理
    views          页面视图

backend
  app
    api
      v1
        endpoints  路由入口
    core           配置、鉴权、异常处理
    db             数据库连接
    models         ORM 模型
    schemas        请求响应模型
    services       业务逻辑层
    tasks          定时任务与异步任务
    utils          公共工具

sql
  schema.sql       建表脚本

docs
  开发脚手架方案.md
  数据库设计.md
```

## 3. 开发阶段建议

### 第一阶段
- 完成登录认证
- 完成用户、课程、成绩、培养方案的基础 CRUD
- 打通学生首页数据汇总接口

### 第二阶段
- 完成学业预警规则与预警记录模块
- 完成知识文档录入和检索接口
- 打通智能问答服务接口

### 第三阶段
- 完成微专业推荐和目标岗位能力分析
- 完成日志、监控、测试和部署脚本

## 4. 接口分层建议

- `api` 负责接收请求和返回响应
- `services` 负责业务编排
- `models` 负责数据库实体定义
- `schemas` 负责请求响应校验
- `tasks` 负责预警扫描、知识库索引等后台任务

## 5. 首批建议开发的页面

- 登录页
- 学生首页
- 成绩查询页
- 培养方案页
- 学业预警页
- 智能问答页
- 微专业推荐页
- 管理员知识库管理页
