# 数据库 E-R 设计

## 1. 设计说明

数据库围绕“用户、学生学业、培养方案、预警、推荐、知识库、问答记录”七类核心业务构建。设计原则如下：

- 满足基础教务查询与智能体调用的数据要求。
- 核心业务数据结构化，便于规则计算与统计分析。
- 预留知识库与推荐模块扩展能力。
- 尽量保持实体职责单一，降低后续维护成本。

## 2. 核心实体

### 2.1 用户实体 `users`
用于保存系统登录账号及角色信息。

关键字段：
- `id`
- `username`
- `password_hash`
- `role`
- `real_name`
- `student_no`
- `teacher_no`
- `college`
- `major`
- `grade`

### 2.2 学生画像实体 `student_profiles`
用于保存学生学业画像与统计信息。

关键字段：
- `user_id`
- `training_program_id`
- `target_career`
- `total_credits_required`
- `earned_credits`
- `failed_course_count`
- `gpa`

### 2.3 课程实体 `courses`
用于保存课程基础信息。

关键字段：
- `course_code`
- `course_name`
- `course_type`
- `credit`
- `hours`
- `college`

### 2.4 开课计划实体 `course_offerings`
用于描述某学期课程开设安排。

关键字段：
- `course_id`
- `semester`
- `teacher_name`
- `weekday`
- `start_section`
- `end_section`
- `location`

### 2.5 学生选课成绩实体 `enrollments`
用于保存学生某门课某学期的选课和成绩结果。

关键字段：
- `student_id`
- `offering_id`
- `semester`
- `score`
- `gpa_point`
- `is_passed`
- `is_retake`

### 2.6 培养方案实体 `training_programs`
用于保存专业年级对应的培养方案。

关键字段：
- `major`
- `grade`
- `required_total_credits`
- `required_mandatory_credits`
- `required_elective_credits`
- `graduation_requirements`

### 2.7 培养方案课程要求实体 `program_course_requirements`
用于保存培养方案中的课程要求明细。

关键字段：
- `training_program_id`
- `course_id`
- `requirement_type`
- `min_credit`
- `is_required`

### 2.8 学业预警规则实体 `warning_rules`
用于保存预警规则配置。

关键字段：
- `rule_name`
- `warning_type`
- `rule_expression`
- `warning_level`
- `is_active`

### 2.9 学业预警记录实体 `warning_records`
用于保存具体预警结果。

关键字段：
- `student_id`
- `rule_id`
- `warning_type`
- `warning_level`
- `reason`
- `suggestion`
- `status`

### 2.10 微专业实体 `micro_majors`
用于保存微专业基础信息。

关键字段：
- `name`
- `direction`
- `description`
- `required_credits`
- `eligibility_rules`

### 2.11 微专业课程关联实体 `micro_major_courses`
用于描述微专业需要修读的课程。

### 2.12 推荐记录实体 `recommendation_records`
用于保存系统生成的课程、微专业和能力建议结果。

### 2.13 知识文档实体 `knowledge_documents`
用于保存教务制度、培养方案解读、办事流程等知识库文档。

### 2.14 问答记录实体 `chat_records`
用于保存用户与智能体的交互内容。

## 3. 实体关系

### 3.1 主要关系说明
- 一个 `users` 可以对应一个 `student_profiles`。
- 一个 `training_programs` 可以关联多个 `program_course_requirements`。
- 一个 `courses` 可以被多个 `course_offerings` 引用。
- 一个学生可以拥有多条 `enrollments` 记录。
- 一个学生可以拥有多条 `warning_records`。
- 一个学生可以拥有多条 `recommendation_records`。
- 一个 `micro_majors` 可以关联多门 `courses`。
- 一个用户可以产生多条 `chat_records`。

## 4. E-R 文字图

```text
users 1 ---- 1 student_profiles
users 1 ---- n enrollments
users 1 ---- n warning_records
users 1 ---- n recommendation_records
users 1 ---- n chat_records

training_programs 1 ---- n program_course_requirements
training_programs 1 ---- n student_profiles

courses 1 ---- n course_offerings
courses 1 ---- n program_course_requirements
courses n ---- n micro_majors   (through micro_major_courses)

course_offerings 1 ---- n enrollments
warning_rules 1 ---- n warning_records
micro_majors 1 ---- n recommendation_records
knowledge_documents 1 ---- n chat_records
```

## 5. 建模补充说明

- 问答记录中的来源信息不直接强依赖某一张知识文档表，可通过 JSON 保存引用来源列表，便于兼容多文档检索结果。
- 预警规则中的 `rule_expression` 可采用简单 DSL、JSON 条件或代码映射标识，便于后端解释执行。
- 推荐记录中的推荐内容可使用 JSON 保存，以兼容课程推荐、微专业推荐、技能推荐三类不同结构。
